# OpenSpec Capability Specs

This directory contains As-Is capability specs for meminsight.

## Start here

1. 00-capability-index.md
2. 13-implementation-parity-matrix.md
3. ../architecture/00-baseline-architecture.md (system-level architecture baseline)

## How to use

1. Identify impacted capability IDs before implementation or review.
2. Validate behavior and tests against the capability acceptance behavior.
3. Update capability docs when baseline behavior intentionally changes.
4. Use architecture docs for subsystem/runtime/dependency context before drafting larger changes.

## Policy

Specs in this directory describe verified current behavior only.
