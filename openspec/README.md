# OpenSpec Directory

This directory is the requirements and behavior governance layer for meminsight.

## Contents

- config.yaml: project and agent configuration for OpenSpec workflow.
- specs/: capability-level As-Is behavior documentation.
- changes/: active delta specs for intentional behavior changes.

## Usage

1. Read specs before code changes.
2. Use changes for planned behavior deltas.
3. Keep specs and implementation in parity.

## Important rule

In this repository, OpenSpec artifacts are the primary source of truth for behavior requirements.
